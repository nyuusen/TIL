# plan-pr-comment skill

以下の手順に従い、プルリクエストのレビューコメントの修正計画・実装を行ってください。

## 手順

1. 現在チェックアウトしているブランチのプルリクエストを特定してください。もし、プルリクエストが見つからない場合は、ユーザーにプルリクエスト番号を質問してください。

2. 特定したプルリクエストについているレビューコメントで、Resolveされていないコメントを抽出してください。

3. レビューコメントそれぞれに対して、以下の「方針・ルール」に従い、以下の修正計画を立ててください。なお、修正計画は、以下の「修正計画フォーマット」に沿ってマークダウンファイルを作成し、ユーザーに提示してください。

## 方針・ルール

- プロジェクト概要やコーディングルール等は、[AGENTS.md](/batch/AGENTS.md) を参照すること
- コメントで指摘されている箇所が、既存のコードの調査を行うこと
  - 既存のコードの調査には、`batch-specify-existing-code`スキルを活用して、対象ファイル・コードの特定を行うこと
- 必要に応じて、以下のフォルダに格納されている仕様書・実装計画を参照すること（ファイル名に、バッチ名（英語 or 日本語）が含まれている）
  - [specs](/batch/specs/)
  - [plan](/batch/plan/)
- 1つのコメントにつき、以下のフォーマットを使用してください。例えば、3つのコメントが存在する場合は、1つのマークダウンファイルの中に以下のフォーマットが3つ記載される内容になります。
- 修正案は、どの案が最も推奨されるかを、以下それぞれの観点から提示するようにしてください。
  - 観点1: 既存コードの挙動・仕様を忠実に再現する場合はどの案が良いか
  - 観点2: 一般論的に、バグ防止・性能面などで評価する場合はどの案が良いか
- 既存のコード（PHP/Laravel）の仕様・挙動を記載するにあたり、ユーザーはPHP/Laravelの構文の知識がないことを前提として、詳細を記述するようにしてください。

## 修正計画フォーマット

```
## コメント1: コメントの概要(タイトル)

- レビューコメント内容
  - 該当箇所（ファイルパスと行番号）
  - 問題点の詳細
  - 現在の仕様・挙動はどうなっているか
- 既存コードの調査結果
  - 既存のコードの該当箇所（ファイルパスと行番号）
  - 仕様・挙動はどうなっているか
- 現在のコードと既存のコードの差分（仕様や挙動）
- 修正案
```
