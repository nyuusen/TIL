# コンテナセキュリティ コンテナ化されたアプリケーションを保護する要素技術

### コンテナへのシークレットの受け渡し

- コンテナイメージに埋めるのはビルド時にシークレットをハードコードする必要があるので良くない
  - シークレットの更新にもイメージ再ビルドが必要になってしまう
- 環境変数での受け渡しも良くない
  - クラッシュ時のデバックログに含まれたり、docker image inspectコマンドで確認可能
- マウントしたボリュームのファイルで受け渡すのが良い
  - tmpfsという仮想ファイルシステムに置かれる
    - これはメモリ上なのでコンテナ停止すれば消えるので安全
  - docker composeでは、tmpfsフィールドを使用する
    ```
    services:
      web:
        image: nginx:alpine
        tmpfs:
          - /app/cache:size=128M,mode=1777
    ```
- ネットワークインターフェイス越しの取得も安全
