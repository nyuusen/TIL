# DELETEとTRUNCATEの処理の違い

- DELETE
  - データの中身を消すDML
  - ディスク上はすぐには消えずフラグを立てる論理削除を行う
  - Undo/Redoログを吐く（大量のI/Oを発生）
    - Undo: 処理前に戻る（ロールバック）
    - Redo: 実行した処理を記録する、障害発生時に障害発生前まで処理をやり直す（ロールフォワード）
  - AUTO_INCREMENTは維持される
  - ディスク空き容量は増えない
    - 実際ファイルに書かれたデータには削除フラグが立ち、そのデータを見ているトランザクションが全くいなくなると、裏側でパージスレッドという掃除屋が動き出す
    - そうなると、物理的に取り除かれて、ページ内のその場所は空き領域としてリスト登録される（OSから見るとファイルサイズは減っていない）
- TRUNCATE
  - DDL
  - ロールバック不可
  - テーブルに紐づいているibdファイルなどを一旦破棄して空の状態で作り直す
  - AUTO_INCREMENTリセット
  - ディスク空き容量は増える
